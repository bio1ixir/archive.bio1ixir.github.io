Contributors Manual


Post: If you got some good stuff that you wish to post on Bio1ixir, you must mail us a .md file with front matter as specified in the archetypes of the organisation repository.

Note: You can write a post in collaboration with atmost one co-author. In that case both of your names, with & separating them, must be mentioned in the YAML front matter of that .md file. If you’re new to Markdown, it’s not scary, it’s very simple and they even provide a cheatsheet which you can find here.

Alternatively, you can create a PR at bio1ixir/blog

Project: If you want to contribute to an ongoing project, you can contact respective Project Manager.

Seed: You can mail us suggesting the topics on which you may want to read. Mailing details are at the bottom.

GitHub:

  1. Fork the repo.
  2. Create Git branch and commit your changes.
  3. Create pull request.
  4. Wait for code reviews. In the mean time, read biology.

While contributing code or raising issues, follow repository’s template only (if provided).

Mailing Details

  • For outsiders: bio1ixir.iitj@gmail.com
  • For members: bio1ixir@googlegroups.com

Note: We are a community in science. Sharing authentic information should be our utmost concern. Subtulism is not an option.
While stating some dynamic stats in your post, that are subjected to changes with time, you must clearly mention the date/year till which the stats held truth. Moderation team will take care that every post with dynamic stats should stand this condition.
Having said that, I encourage you to read Founder's Words before making your next move.
